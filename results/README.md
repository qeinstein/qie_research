# Results Directory

This directory contains tracked scientific outputs intended for review and manuscript use.

- `metrics/` holds compact metric summaries, NTK summaries, statistical tests, and aggregated CSV outputs.
- Add other compact result artifacts under `results/` only when a run produces outputs that are worth tracking in Git.
- Keep heavyweight runtime, profiling, and evaluation logs out of Git and link them from records or Pull Requests when relevant.

If a result supports a paper claim, it must be reproducible from versioned code and configuration and referenced from a repository record or Pull Request.
