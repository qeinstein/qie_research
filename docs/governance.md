# Governance Documents

This file summarizes the governance documents now stored under `docs/governance/`.

- `docs/governance/logging_protocol.md` defines how GitHub Issues are used for milestones, phase outcomes, meetings, decisions, and risks.
- `docs/governance/semantic_commit_guide.md` defines commit message conventions.
- `docs/governance/phase_outcome_matrix.md` defines how each research phase is verified.
- `docs/governance/peer_review_protocol.md` defines Pull Request and teammate review requirements.
- `docs/governance/reference_library_maintenance.md` defines citation-library and PDF-management rules.

If a process matters repeatedly, document it in `docs/governance/` rather than only in chat or memory.
