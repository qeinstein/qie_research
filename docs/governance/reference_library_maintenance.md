# Reference Library Maintenance Protocol

## Canonical Files

- Canonical bibliography: `references/bibtex/project_library.bib`
- Reading notes and PDF source details: `references/reading_notes/`
- Screening decisions: record in `references/reading_notes/` unless the literature review grows large enough to justify a separate screening log.

## Citation Key Standard

Use deterministic citation keys:

```text
SurnameYearShortTitle
```

Examples:

- `schuld2021quantumkernels`
- `rahimi2007randomfeatures`
- `jacot2018ntk`

Pick one casing convention and keep it stable. Do not rename existing keys casually because manuscript citations, notes, and review comments will break.

## PDF Naming Standard

If you keep local PDFs outside the repository, store them as:

```text
FirstAuthorYear-ShortTitle.pdf
```

Examples:

- `Schuld2021-QuantumKernels.pdf`
- `Rahimi2007-RandomFeatures.pdf`

Do not rely on committed PDF copies. Record each paper's source, DOI, and access instructions in `references/reading_notes/`.

## Merge-Conflict Prevention Rules

- Make bibliography-only changes in dedicated `ref/` branches whenever possible.
- Do not reorder the entire BibTeX file unless the Pull Request exists solely to normalize the library.
- Add one entry block at a time and keep nearby formatting unchanged.
- Rebase before merge if another bibliography Pull Request landed first.
- Never mix large manuscript rewrites with large bibliography cleanups.

## Required Steps for Adding a Reference

1. Add the BibTeX entry to `project_library.bib`.
2. Record the PDF source or local access instructions in `references/reading_notes/`.
3. Add a reading note file or extend an existing note when the paper materially affects the project.
4. Mention the citation key in the Pull Request description.
5. If the paper changes project direction, open a decision record.

## Required Steps for Editing an Existing Entry

1. Preserve the citation key unless there is a strong reason to change it.
2. Explain the edit in the Pull Request.
3. Search manuscript and notes for downstream key usage before renaming anything.

## Bibliography Quality Rules

- Prefer DOI-backed or publisher-exported BibTeX over low-quality autogenerated entries.
- Normalize author names, titles, venues, and years for consistency.
- Remove duplicate entries immediately once discovered.
- Record uncertain metadata in comments rather than guessing silently.

## Team Coordination Rule

Only one bibliography-heavy Pull Request should be in final review at a time. This keeps merge conflicts in `project_library.bib` manageable and avoids broken citation keys across manuscript branches.
